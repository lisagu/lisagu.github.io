git 分布式 多镜像 用户密码的存储更新

发表于 2020-01-14

windows git bash 密码存储

选择一个就近的镜像服务器地址（非主主服务器）clone仓库到本地，首次会要求输入用户名密码，window系统这个会被记录到
‘user account’(控制面板中打开) -> ‘window 凭证’

这时会看到两个两个凭证，一个https://镜像地址，一个包含https://username@镜像地址
当推送变动到远程仓库，这时候会要去输入用户名密码，这是争对主镜像的。(个人觉得很反人类)

这个时候在’window凭证’中可以看到又保存了上述类似的两条。

记住密码后，下次操作不在需要输入。

当密码更新后 需要手动在window

凭证中修改这里被记住的此仓库关联的所以密码，可以选择修改，或者直接删除再次git操作会要求重新输入。

由此推测，clone等下拉是从选择的镜像下载的代码，但是push等上推操作是对主镜像的，镜像间的同步由主镜像到各个副镜像。
